This HTML file is a template. You can add webfonts, meta tags, or analytics to this file. The build step will place the bundled scripts into the body tag. In this template, you are free to use Development Tool's tags. MDT application will replace these tags with related data of this Development Tool project, which is found in dt_config.json file.


Here are all available Development Tool's tags:


  • < dt-title > This tag's content is replaced with Development Tool's title.

  • < dt-icon > This tag's content is replaced with Development Tool's icon file name.

  • < dt-index > This tag's content is replaced with Development Tool's index file name.

  • < dt-body-page > This tag's content is replaced with Development Tool's body_page file name.

  • < dt-footer-page > This tag's content is replaced with Development Tool's footer_page file name.

  • < dt-version > This tag's content is replaced with Development Tool's version.

  • < dt-author > This tag's content is replaced with Development Tool's author name.

  • < dt-fail > This tag's content displays only when Development Tool's tags fail to load.

  • < dt-success > This tag's content displays only when Development Tool's tags are loaded.

You can remove the file type signature by applying an empty attribute "only-name".
for example, < dt-index only-name > removes the first signature ".html" found at the very end of the string.


Since the content of Development Tool's tags are going to be replaced, it can be useful to initiate their content to setup a default value for the tags in case they fail to load.
